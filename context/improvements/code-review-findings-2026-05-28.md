## Priority 1 — Must Fix (correctness, security, or data integrity at risk)

- [ ] **Orphaned vectors accumulate on partial failure** — when `client.upsert()` succeeds but stale-vector `client.delete()` fails, the fingerprint is not updated so the file is reindexed next cycle with a new `index_version` UUID. The old orphans are never cleaned. Under repeated Qdrant degradation events this permanently bloats the index. Add reconciliation or make the delete idempotent by `index_version`. — `ingest/index_documents.py:109–138`

- [ ] **`logging.basicConfig` at module import level** — runs when any code imports `indexer.watcher`, reconfiguring the root logger for all importers including tests. Move into `main()`. — `indexer/watcher.py:43–46`

- [ ] **External mutation of private module state** — `index_documents._collection_ensured = False` in `reset_collection.py` reaches into another module's private state. Breaks silently at runtime on any rename or encapsulation refactor. Expose `reset_collection_state()` on `index_documents` and remove the external mutation. — `ingest/reset_collection.py:32`, `ingest/index_documents.py:45`

- [ ] **Suppressed WAL failure is completely invisible** — if `PRAGMA journal_mode=WAL` fails (read-only mount, network filesystem), the `suppress` block swallows it with no log output. Add `logger.warning()` inside the suppress block. — `common/sqlite_store.py:21–22`

## Priority 2 — Should Fix (maintainability, hidden bugs, technical debt)

- [ ] **Name collision: `get_hash` in two unrelated modules** — `user_store.get_hash` (password hashing) and `fingerprint_store.get_hash` (file fingerprinting) have identical names, different semantics. Rename to `get_password_hash` and `get_file_hash` respectively. — `web/user_store.py:37`, `indexer/fingerprint_store.py:30`

- [ ] **Confusing public/private naming on document removal functions** — `delete_document` (Qdrant-only) and `remove_indexed_document` (vectors + fingerprint) have overlapping names with no clear distinction. Rename: `_delete_vectors` (private, no external callers outside tests) and `remove_document`. — `ingest/index_documents.py:187,196`

- [ ] **No per-request audit trail for `ALLOW_INSECURE_LOCALONLY` bypass** — only the startup warning is logged; unauthenticated requests through the bypass leave no trace. Log at the per-request level when auth is bypassed. — `web/api_server.py:189`

- [ ] **`KeywordIndex._build` is 85 lines with 3-level nesting** — exceeds the project's ~50-line function limit. Scroll + tokenization + threshold logic should be extracted into `_scroll_and_tokenize()`. — `api/keyword_index.py:84–168`

- [ ] **`KEYWORD_SEARCH_ENABLED` checked inside the Qdrant scroll loop** — when disabled, the full collection scroll still runs. Move the check before the loop. — `api/keyword_index.py:116–117`

- [ ] **Hours-to-seconds conversion duplicated** — `expiry_hours * 3600` appears independently in `user_store.py:78` and `api_server.py:462`. Add `SESSION_EXPIRY_SECONDS` to `settings.py` and remove both raw `* 3600` expressions.

- [ ] **`_wait_for_capacity` / `_acquire_rag_capacity` split creates inconsistent error contracts** — `_run_rag_with_timeout` gets `HTTPException` on timeout; `_rag_stream_response` gets raw `TimeoutError`. Merge into one function. — `web/api_server.py:255–267`

- [ ] **Hand-rolled timing dict in `index_file` doesn't respect `RAG_TIMING`** — replace with `api/timing.py:timed()` context managers for consistency with the retrieval pipeline. — `ingest/index_documents.py:146–180`

- [ ] **Retrieval internals (`Chunk`, `rerank`) imported into the web layer for warmup** — `api_server.py` constructs a synthetic `Chunk` and calls `rerank()` directly. Expose `warmup_reranker()` from `api.retrieval` instead. — `web/api_server.py:37,503–507`

- [ ] **`web/auth.py` is a dead abstraction** — three delegating functions that add no logic, and the abstraction is already broken since `api_server.py` imports `user_store` directly for `init_db` and `purge_expired_sessions`. Delete the module and inline its calls. — `web/auth.py`

- [ ] **Copy-pasted `init_db` pattern across three modules** — `common/index_state.py:11`, `indexer/fingerprint_store.py:17`, and `web/user_store.py:18` each open a connection and run `CREATE TABLE IF NOT EXISTS` with no shared base. Add `init_tables(ddl: str)` helper to `SqliteStore`.
